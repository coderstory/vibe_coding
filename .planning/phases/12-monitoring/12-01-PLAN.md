# Phase 12: RocketMQ 监控面板

**Phase:** 12
**Name:** 监控面板
**Goal:** 实现集群监控、Broker 状态、堆积监控和实时图表
**Created:** 2026-04-29

---

## 1. Success Criteria

- [ ] 用户可以看到集群概览（Broker数量、Topic数量、Consumer数量、堆积总量）
- [ ] 用户可以查看各 Broker 的状态（在线/离线、版本、运行时间）
- [ ] 用户可以查看各 Topic 的消息堆积量
- [ ] 用户可以查看实时 QPS 监控图表（生产/消费）
- [ ] 用户可以查看延迟统计（发送延迟、消费延迟）

---

## 2. Tech Stack

| 类别 | 技术 | 说明 |
|------|------|------|
| 前端图表 | ECharts | via vue-echarts |
| 实时刷新 | 轮询 | 每 5 秒轮询一次 |
| 后端 API | Spring Boot | REST API |
| MQ 管理 | DefaultMQAdminExt | RocketMQ 5.3.2 |

---

## 3. 后端实现

### 3.1 集群概览 API

**Endpoint:** `GET /api/rocketmq/dashboard/overview`

### 3.2 Broker 状态列表 API

**Endpoint:** `GET /api/rocketmq/dashboard/brokers`

### 3.3 Topic 堆积量 API

**Endpoint:** `GET /api/rocketmq/dashboard/topics`

### 3.4 Broker 运行时指标 API

**Endpoint:** `GET /api/rocketmq/dashboard/broker/{brokerName}/metrics`

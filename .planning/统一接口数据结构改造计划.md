# 前后端接口数据结构统一改造计划

## 目标

统一前后端 API 响应数据结构，消除当前 `Map<String, Object>` 手动拼接和独立 axios 实例的混乱模式。

## 统一数据结构

```json
{
  "code": 200,
  "message": "success",
  "data": { ... }
}
```

## 后端改造

### 改造范围

| 文件 | 当前模式 | 目标模式 |
|------|---------|---------|
| AuditLogController.java | `ResponseEntity<Map<String, Object>>` | `ResponseEntity<ApiResponse<T>>` |
| KnowledgeController.java | `ResponseEntity<Map<String, Object>>` + 私有 ok()/fail() | `ResponseEntity<ApiResponse<T>>` |
| MenuController.java | `ResponseEntity<Map<String, Object>>` | `ResponseEntity<ApiResponse<T>>` |
| RoleController.java | `ResponseEntity<Map<String, Object>>` | `ResponseEntity<ApiResponse<T>>` |

### 1. AuditLogController.java 改造

**文件**: `springboot/src/main/java/cn/coderstory/springboot/controller/AuditLogController.java`

**变更内容**:
- 导入 `cn.coderstory.springboot.vo.ApiResponse`
- 方法返回类型从 `ResponseEntity<Map<String, Object>>` 改为 `ResponseEntity<ApiResponse<...>>`
- 删除手动 HashMap 构建，改用 `ApiResponse.success(data)`

**改造后代码**:
```java
@GetMapping("/logs")
public ResponseEntity<ApiResponse<Map<String, Object>>> getAuditLogs(
        @RequestParam(required = false) String operator,
        @RequestParam(required = false) String operationType,
        @RequestParam(required = false) @DateTimeFormat(pattern = "yyyy-MM-dd HH:mm:ss") LocalDateTime startTime,
        @RequestParam(required = false) @DateTimeFormat(pattern = "yyyy-MM-dd HH:mm:ss") LocalDateTime endTime,
        @RequestParam(defaultValue = "1") Integer page,
        @RequestParam(defaultValue = "20") Integer size) {

    Page<AuditLog> pageParam = new Page<>(page, size);
    var result = auditService.getAuditLogPage(pageParam, operator, operationType, startTime, endTime);

    Map<String, Object> data = new HashMap<>();
    data.put("records", result.getRecords());
    data.put("total", result.getTotal());
    data.put("size", result.getSize());
    data.put("current", result.getCurrent());
    data.put("pages", result.getPages());

    return ResponseEntity.ok(ApiResponse.success(data));
}
```

### 2. KnowledgeController.java 改造

**文件**: `springboot/src/main/java/cn/coderstory/springboot/controller/KnowledgeController.java`

**变更内容**:
- 导入 `cn.coderstory.springboot.vo.ApiResponse`
- 删除私有 `ok()` 和 `fail()` 方法
- 所有方法返回类型改为 `ResponseEntity<ApiResponse<...>>`
- 使用 `ApiResponse.success(data)` 和 `ApiResponse.error(message)`

**改造后代码**:
```java
// 删除这两个私有方法
// private ResponseEntity<Map<String, Object>> ok(Object data) { ... }
// private ResponseEntity<Map<String, Object>> fail(String message) { ... }

// 示例方法改造
@GetMapping("/categories/tree")
public ResponseEntity<ApiResponse<List<KnowledgeCategory>>> getCategoryTree() {
    List<KnowledgeCategory> tree = knowledgeService.getCategoryTree();
    return ResponseEntity.ok(ApiResponse.success(tree));
}

@DeleteMapping("/categories/{id}")
public ResponseEntity<ApiResponse<Void>> deleteCategory(@PathVariable Long id) {
    boolean success = knowledgeService.deleteCategory(id);
    if (success) {
        return ResponseEntity.ok(ApiResponse.success());
    } else {
        return ResponseEntity.ok(ApiResponse.error("删除失败"));
    }
}
```

## 附录: ApiResponse 常用方法

```java
// 成功响应
ApiResponse.success()                          // { code: 200, message: "success", data: null }
ApiResponse.success(T data)                     // { code: 200, message: "success", data: data }
ApiResponse.success("操作成功", T data)         // { code: 200, message: "操作成功", data: data }

// 错误响应
ApiResponse.error("错误信息")                    // { code: 500, message: "错误信息", data: null }
ApiResponse.error(400, "参数错误")              // { code: 400, message: "参数错误", data: null }
ApiResponse.badRequest("请求参数错误")           // { code: 400, message: "请求参数错误", data: null }
ApiResponse.unauthorized("未登录")               // { code: 401, message: "未登录", data: null }
ApiResponse.forbidden("没有权限")                // { code: 403, message: "没有权限", data: null }
ApiResponse.notFound("资源不存在")               // { code: 404, message: "资源不存在", data: null }
```

# 实施者子代理提示模板

调度实施者子代理时使用此模板。

```
Task 工具（通用）：
  description: "实施任务 N：[任务名称]"
  prompt: |
    你正在实施任务 N：[任务名称]

    ## 任务描述

    [计划中任务的完整文本 - 粘贴到这里，不要让子代理读取文件]

    ## 上下文

    [场景设置：这适合哪里、依赖关系、架构上下文]

    ## 开始之前

    如果你对以下方面有疑问：
    - 需求或验收标准
    - 方法或实施策略
    - 依赖关系或假设
    - 任务描述中的任何不清楚之处

    **现在就问。** 在开始工作前提出任何顾虑。

    ## 你的工作

    一旦你清楚需求：
    1. 准确实施任务指定的内容
    2. 编写测试（如果任务说要遵循 TDD）
    3. 验证实现工作
    4. 提交你的工作
    5. 自我审查（见下文）
    6. 报告返回

    工作目录：[目录]

    ## 自我审查

    在报告返回之前，检查：
    - [ ] 实施了任务指定的所有内容
    - [ ] 测试通过（如果编写了测试）
    - [ ] 没有添加未请求的功能
    - [ ] 遵循项目约定
    - [ ] 提交消息清晰

    ## 报告格式

    返回：
    1. 你实施了什么（摘要）
    2. 测试结果（如果运行）
    3. 任何问题或决策
    4. 任何需要审查的问题
```

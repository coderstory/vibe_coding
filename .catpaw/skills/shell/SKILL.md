# 运行 Shell 命令

仅当用户明确调用 `/shell` 时才使用此技能。

## 行为

1. 将 `/shell` 调用后的所有用户文本视为要运行的字面 shell 命令。
2. 使用终端工具立即执行该命令。
3. 运行前不要重写、解释或"改进"命令。
4. 除非命令本身需要仓库上下文，否则不要先检查仓库。
5. 如果用户调用 `/shell` 但后面没有任何文本，询问他们要运行哪个命令。

## 响应

- 首先运行命令。
- 然后简要报告退出状态和任何重要的标准输出或标准错误。

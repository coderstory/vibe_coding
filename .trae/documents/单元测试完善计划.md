# 单元测试完善计划

> 目标：执行全部单元测试，修复编译错误和测试错误，确保测试数据正确清理

## 问题分析

1. **测试需要访问实际中间件**：不能 mock Redis 和 RocketMQ
2. **测试数据未清理**：导致重复执行时出现数据冲突
3. **编译错误**：需要检查并修复

## 实施步骤

### 步骤 1：检查现有测试文件

检查以下测试文件的状态：
- `SignServiceTest.java`
- `StockServiceTest.java`
- `DistributedLockServiceTest.java`
- `OrderServiceTest.java`
- `IdempotentServiceTest.java`
- `QpsLimiterTest.java`

### 步骤 2：修复测试数据清理问题

#### 2.1 添加 `@AfterEach` 或 `@AfterAll` 清理方法

```java
@AfterEach
void tearDown() {
    // 清理 Redis 测试数据
    redisTemplate.delete("test:key:*");
    // 清理数据库测试数据
    orderMapper.delete(lambdaQuery().eq(Order::getOrderNo, "TEST_XXX"));
}
```

#### 2.2 使用测试数据库

```yaml
# application-test.yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/admin_system_test
```

### 步骤 3：修复编译错误

检查并修复：
1. 导入路径错误
2. 方法签名不匹配
3. 类型转换问题

### 步骤 4：运行测试验证

```bash
cd springboot
.\mvnw.cmd test
```

## 测试数据清理策略

### Redis 数据清理
- 使用通配符删除：`redisTemplate.delete(keys)`
- 清理前缀：`seckill:*` 测试相关数据

### MySQL 数据清理
- 使用 `@AfterEach` 清理测试插入的数据
- 测试订单号使用唯一前缀：`TEST_` + 时间戳

## 预期结果

1. ✅ 所有测试编译通过
2. ✅ 所有测试执行成功
3. ✅ 测试数据正确清理
4. ✅ 重复执行无数据冲突
